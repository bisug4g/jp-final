# 📋 Environment Variables & Secrets for Railway Deployment

This document provides a complete reference for configuring environment variables in Railway for the JP (Jayti's Personal) application.

## 🚨 Important Security Notes

- **NEVER commit secrets to the repository**
- Store all sensitive credentials in Railway's environment variables
- Use Railway's reference syntax for auto-generated values
- Rotate keys and secrets regularly

---

## 🗄️ Database (PostgreSQL)

These variables are **auto-generated by Railway** when you add a PostgreSQL service. Use Railway's reference syntax to link them:

```bash
# Auto-generated by Railway - Reference from Postgres service
DATABASE_URL=${{ Postgres.DATABASE_URL }}
PGHOST=${{ Postgres.RAILWAY_PRIVATE_DOMAIN }}
PGPORT=5432
PGUSER=${{ Postgres.POSTGRES_USER }}
PGPASSWORD=${{ Postgres.POSTGRES_PASSWORD }}
PGDATABASE=${{ Postgres.POSTGRES_DB }}
```

**Setup:**
1. Add PostgreSQL plugin in Railway
2. Add the above variables to your web service
3. Railway will automatically resolve the references

## 🔐 Django Core Settings

### SECRET_KEY (Required)
**Description:** Django secret key for cryptographic signing
**How to generate:**
```bash
python -c 'from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())'
```
**Example:**
```bash
SECRET_KEY=django-insecure-xj8r_k#p9m-vz@4n$q2w!e5*h6&u7y-t3g9b1c0d2f8
```

### DEBUG (Required)
**Description:** Debug mode (MUST be False in production)
**Value:**
```bash
DEBUG=False
```

### TIME_ZONE
**Description:** Server timezone
**Default:** `Asia/Kolkata`
```bash
TIME_ZONE=Asia/Kolkata
```

### RAILWAY_DEBUG
**Description:** Railway-specific debug flag
**Value:**
```bash
RAILWAY_DEBUG=false
```

## 🌐 Domain & CORS Configuration

### ALLOWED_HOSTS (Required)
**Description:** Comma-separated list of allowed domains
**Value:**
```bash
ALLOWED_HOSTS=localhost,127.0.0.1,jaytibirthday.in,www.jaytibirthday.in,.railway.app,.emergentagent.com,.emergentcf.cloud,.emergent.host
```

### DEPLOYMENT_URL
**Description:** Full URL of your deployment
**Example:**
```bash
DEPLOYMENT_URL=https://jaytibirthday.in
```

### CSRF_TRUSTED_ORIGIN
**Description:** Trusted origin for CSRF protection
**Example:**
```bash
CSRF_TRUSTED_ORIGIN=https://jaytibirthday.in
```

## 🔥 Firebase Configuration (For Push Notifications)

**⚠️ All Firebase credentials are SECRETS - DO NOT commit to repository**

### Client-Side Firebase Config (Web App)

```bash
FIREBASE_API_KEY=AIzaSyXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX
FIREBASE_AUTH_DOMAIN=your-project.firebaseapp.com
FIREBASE_PROJECT_ID=your-project-id
FIREBASE_STORAGE_BUCKET=your-project.appspot.com
FIREBASE_MESSAGING_SENDER_ID=123456789012
FIREBASE_APP_ID=1:123456789012:web:abcdef1234567890
FIREBASE_MEASUREMENT_ID=G-XXXXXXXXXX
```

**How to get:** Firebase Console → Project Settings → General → Your apps → Web app

### Server-Side Firebase Admin SDK (Service Account)

```bash
FIREBASE_PRIVATE_KEY_ID=abc123def456
FIREBASE_PRIVATE_KEY=-----BEGIN PRIVATE KEY-----\nMIIEvQIBADANBgkq...\n-----END PRIVATE KEY-----\n
FIREBASE_CLIENT_EMAIL=firebase-adminsdk-xxxxx@your-project.iam.gserviceaccount.com
FIREBASE_CLIENT_ID=123456789012345678901
FIREBASE_AUTH_URI=https://accounts.google.com/o/oauth2/auth
FIREBASE_TOKEN_URI=https://oauth2.googleapis.com/token
FIREBASE_AUTH_PROVIDER_CERT_URL=https://www.googleapis.com/oauth2/v1/certs
FIREBASE_CLIENT_CERT_URL=https://www.googleapis.com/robot/v1/metadata/x509/firebase-adminsdk-xxxxx%40your-project.iam.gserviceaccount.com
```

**How to get:**
1. Firebase Console → Project Settings → Service Accounts
2. Click "Generate new private key"
3. Download JSON file
4. Extract individual fields and add to Railway

**⚠️ Note:** For `FIREBASE_PRIVATE_KEY`, replace actual newlines with `\n` in Railway

## 💾 Google Drive Storage (For Audio Recordings)

**Purpose:** Store 1-hour audio chunks to save backend server space

### Option 1: Service Account JSON File
```bash
GOOGLE_DRIVE_CREDENTIALS_PATH=/path/to/service-account-key.json
```

### Option 2: Service Account JSON String (Recommended for Railway)
```bash
GOOGLE_DRIVE_CREDENTIALS_JSON={"type":"service_account","project_id":"...","private_key_id":"...","private_key":"-----BEGIN PRIVATE KEY-----\n...\n-----END PRIVATE KEY-----\n","client_email":"..."}
```

### Folder Configuration

**Option A: Specific Folder ID (Recommended)**
```bash
GOOGLE_DRIVE_FOLDER_ID=1gQu8i7TtVI--EiVIW6N7wCL-GzubKflA
```

**Option B: Folder Name (Will search/create)**
```bash
GOOGLE_DRIVE_FOLDER_NAME=Secret Audio Recordings
```

**Setup Guide:** See `docs/GOOGLE_DRIVE_SETUP.md`

## 🚀 Deployment Checklist

### Before Deploying to Railway:

- [ ] Generate new `SECRET_KEY`
- [ ] Set `DEBUG=False`
- [ ] Configure PostgreSQL service and link variables
- [ ] Add all Firebase credentials from service account JSON
- [ ] Add AI service API keys (Emergent, GitHub Models, Gemini)
- [ ] Generate and add VAPID keys for push notifications
- [ ] Configure Google Drive credentials and folder ID
- [ ] Set domain in `ALLOWED_HOSTS` and `DEPLOYMENT_URL`
- [ ] Verify all secrets are stored in Railway, not in code
- [ ] Test deployment with health check endpoint

### After Deployment:

- [ ] Run migrations: `python manage.py migrate`
- [ ] Create superuser: `python manage.py createsuperuser`
- [ ] Test Firebase push notifications
- [ ] Test audio recording to Google Drive
- [ ] Verify consent flow works correctly
- [ ] Check all AI services are functional

## 🔍 Troubleshooting

### Database Connection Issues
- Verify `DATABASE_URL` is properly linked from Postgres service
- Check PostgreSQL service is running in Railway
- Ensure network policies allow private networking

### Firebase Errors
- Verify `FIREBASE_PRIVATE_KEY` has `\n` for newlines (not actual newlines)
- Check service account has proper permissions
- Ensure project ID matches Firebase console

### Google Drive Upload Failures
- Verify service account JSON is valid
- Check folder ID is correct and shared with service account
- Ensure service account has "Editor" permissions on folder

### AI Service Failures
- Check API keys are valid and have sufficient credits
- Verify base URLs are correct
- Test each service independently

## 🔒 Security Best Practices

1. **Never commit secrets to Git**
   - Use `.gitignore` for `.env` files
   - Store all secrets in Railway's encrypted environment variables

2. **Rotate keys regularly**
   - SECRET_KEY: Every 6 months
   - API keys: Every 3-6 months
   - Service account keys: Every year

3. **Use Railway's built-in secrets**
   - Railway encrypts environment variables at rest
   - Variables are only decrypted at runtime

4. **Limit service account permissions**
   - Google Drive: Only "Editor" on specific folder
   - Firebase: Only required roles for Admin SDK

5. **Monitor API usage**
   - Set up billing alerts for AI services
   - Monitor Firebase usage in console
   - Track Google Drive storage quota
